# casa66 中古家具 AI査定アプリ（石川さん専用PoC）

写真をアップロードするだけで、ブランド・推定定価・買取/メルカリ相場を査定し、
そのままフリマアプリに貼れる**出品説明文**まで作れるツールです。

- **査定エンジンはClaudeが主軸**（`ANTHROPIC_API_KEY` 1個で完結。Vision写真検索＋Web検索で裏取り）
- Geminiエンジン（Cloud Vision併用）も切替で使えます（従来どおり）

## 🚀 かんたん起動（石川さん向け・2ステップ）

**① たいきさんから受け取ったキーを、このフォルダに `api_key.txt` という名前で保存する**

- このフォルダにある `api_key.txt.example` をコピーして、名前を `api_key.txt` に変える
- 中身を、たいきさんから受け取ったキー1行だけに書き換えて保存する
  （`sk-ant-...` で始まる長い文字列です）
- ⚠️ このキーは他の人に渡さないでください。`api_key.txt` は自分のパソコンの中だけに置きます

**② 起動ツールをダブルクリックする**

- **Windowsの方**：`run_furniture_app.bat` をダブルクリック

黒い画面（またはターミナル）が開いて、しばらくするとブラウザで査定アプリが自動で開きます。

> **初回だけ**：必要な部品を自動で準備するので、最初の1回は少し時間がかかります（数分）。2回目以降はすぐ開きます。
> **終わるとき**：ブラウザを閉じたあと、黒い画面で `Ctrl + C` を押すか、そのまま閉じてください。

## 🔧 上級者向け：コマンドで起動する

ターミナルを使い慣れている場合は、従来どおりコマンドでも起動できます。

1. このフォルダをターミナルで開く（`cd` でこのフォルダに移動）
2. 初回だけ依存パッケージを入れる：
   ```
   pip install -r requirements.txt
   ```
3. APIキーを環境変数に設定する（Claudeエンジンを使うなら1個でOK）：
   ```
   export ANTHROPIC_API_KEY="sk-ant-..."
   ```
   （Windowsのコマンドプロンプトなら `set ANTHROPIC_API_KEY=sk-ant-...`）
4. アプリを起動：
   ```
   streamlit run app.py
   ```

## 🔒 APIキーの取り扱いについて（重要）

- APIキーは `api_key.txt`（このフォルダ内・自分のパソコンだけに置くファイル）に保存します。
- **`api_key.txt` は GitHub にもDriveにも同期されません**（`.gitignore` で除外済み）。安心して自分のキーを置けます。
- 起動ツール（`.bat` / `.command`）やアプリ本体には、キーを一切書き込んでいません。
- サンプルの `api_key.txt.example` は「書き方の見本」なので、中身は本物のキーではありません。

## ☁️ クラウド公開（Streamlit Community Cloud・管理者向け）

石川さんが自分のPCを立ち上げなくても、公開URLからブラウザだけで使えるようにする方法です（設定はたいきさんが行います）。

**デプロイ手順の要点:**

1. Streamlit Community Cloud で、このリポジトリの `app.py`（`clients/TEMP/projects/sns-consulting/members/ishikawa/tools/furniture-valuation/app.py`）を指定してデプロイする。
2. アプリの **Settings → Secrets** に、以下の2つを設定する（**値は実際のキー・合言葉に置き換える。ここに本物の値を書く場所で、コードやGitには残りません**）:
   ```toml
   ANTHROPIC_API_KEY = "sk-ant-（実際のClaude APIキー）"
   APP_PASSCODE = "（石川さんとたいきだけが知る合言葉）"
   ```
3. 保存すると再起動し、公開URLで使えるようになります。

**仕組み（コード側の対応）:**

- APIキーは **① `st.secrets`（クラウドのSecrets）→ ② 環境変数** の順で読みます。ローカルでは環境変数（`api_key.txt`経由）、クラウドではSecretsが使われ、**同じコードで両方動きます**。
- `APP_PASSCODE` を設定すると、公開URLを開いた人にまず**合言葉の入力画面**が出ます（正しい合言葉を入れるまで査定画面は表示されません）。たいきさんのAPIキーが不特定多数に使われるのを防ぐためです。
- **ローカル運用では `APP_PASSCODE` を設定しない**ので、合言葉ゲートは出ず、今まで通りダブルクリックで即使えます（UXは変わりません）。
- 合言葉・APIキーの実値はコードには一切書かれていません（Secrets/環境変数からのみ読み込み）。

> ⚠️ Secrets に設定する値（APIキー・合言葉）は関係者以外に共有しないでください。合言葉は石川さんとたいきさんのみで共有します。

## 主な機能

### 1. 査定サマリー（必ず出す4項目）
結果の一番上に、査定で一番見たい**商品名・ブランド・定価額・寸法**の4項目を大きくまとめて表示します。取得できなかった項目は「—」と表示し、勝手に断定しません。定価には出典ページのリンクが付きます。

### 2. AI査定（ブランド・定価・相場）
写真からブランド・商品名・型番を判定し、Web検索で定価の裏取りをしたうえで、業者買取相場・メルカリ相場・おすすめ出品チャネルを提示します。

### 3. 出品説明文の自動生成
査定結果から、**「安心・誠実型」テンプレ**（石川さんが検証済みの勝ちパターン）で出品文を自動生成します。

- メルカリ／ジモティー／ヤフオクの3アプリ分をタブで切替
- 商品名・ブランド・サイズは査定結果から自動で差し込み（コピー用の【】枠は維持）
- 傷写真の番号など、AIでは埋められない箇所は `【◯枚目】` のまま残ります。**投稿前に実物に合わせて差し替えてください**
- 各文面は `st.code` ブロックで表示（右上にコピーアイコンが出ます）

### 4. 着払い可否の自動判定
ゆうパック等の「3辺合計160サイズ」基準に準拠し、以下のルールで小型版／大型版を自動判定します。

- **幅・奥行・高さの3辺合計が160cm以内、かつ重量が25kg以内** → 小型版（引き取り基本＋着払い発送あり）
- それ以外 → 大型版（引き取り限定・着払いなし）
- 3辺合計が **150〜170cmの境界付近** は誤差で判定が変わりうるため、「実測での確認を推奨」の注意が出ます
- 寸法・重量が未確定のときは、安全側（大型＝引き取り限定）を仮表示します

寸法・重量はAIが写真から推定しますが、自信がない場合は空欄（null）になります。その場合や、実測値の方が正確な場合は、画面上の「幅／奥行／高さ／重量」欄に手入力で上書きできます（上書きすると査定サマリー・着払い判定・出品文のサイズ表記すべてに反映されます）。

### 5. 利益シミュレーション
査定結果の下で、チャネル別の**想定手取り**を試算できます。casa66は家具を**無料引き取り**し、**小型は着払い発送（送料は購入者負担）／大型は引き取り限定（送料0）**が基本なので、出品者の送料負担は原則0で計算します。

- **想定売却価格**：初期値はメルカリ相場の中央値。実際に付ける値段に書き換えできます。
- **手数料率（目安）**：メルカリ10%・ジモティー0%（出典: `research-summary.md` 手数料表）／ヤフオク10%（※一般的な個人出品の目安・本アプリの写真研究データ外・要検証）。いずれも「目安」表示で、実売却前に各サービスの最新規約の確認を促します。
- **出品者送料・梱包清掃コスト**：初期値0（無料引き取り・着払い/引き取り前提）。かかった分だけ入力で上書き可。
- 3チャネルを小さな表で並べ、**手取りが最大のチャネルを強調**。手数料0のジモティーが手取り最大になりやすい点、メルカリは集客力が高い点を断定せず注記します。
- 想定売却価格が0のときは「手入力してください」と案内します。

### 6. CSVダウンロード
査定1回分の結果（日時・商品名・ブランド・型番・推定定価・寸法・重量・買取相場・メルカリ相場・着払い可否・おすすめチャネル）を1行のCSVとしてダウンロードできます。ヘッダー行付き・UTF-8 BOM付きなので、Excelでそのまま開いても文字化けしません。石川さんの管理シートに貼り付けてお使いください。

## 撮影のコツ・撮影チェックリスト

アプリ内に2つのガイドがあります（写真アップロード欄の近く）。

- **撮影のコツ**：AI査定の精度を上げるための撮り方（全体・ロゴ/タグ・接合部など）。
- **撮影チェックリスト（早く・高く売るための写真）**：石川さんの写真研究 `../ab-test-proposal/research-summary.md`（2026-05-25調査）準拠。3アプリ横断の共通5原則＋メルカリ／ジモティー／ヤフオクのタブに分け、チェックボックス形式で撮りながら確認できます（チェック状態は保存されません）。
  - メルカリ・ジモティーは research-summary.md 準拠（出典明記）。
  - **ヤフオクは同ファイルに調査項目が無いため、「一般的な中古出品の目安（研究データ外・要検証）」と注記**した汎用項目のみを表示します。

## 既知の制約・今後の余地

- 商品名（product_name）は「一般に通じる商品名・シリーズ名」を、物証（ロゴ・タグ・Web検索の裏取り）がある場合のみ返します。雰囲気だけで有名商品名を断定しない設計です（取得できなければ「—」表示）。
- 利益シミュレーションの手数料率は**目安**です。メルカリ10%・ジモティー0%は research-summary.md 由来、ヤフオク10%は一般的な個人出品の目安（研究データ外・要検証）。実際の売却前に各サービスの最新規約を確認してください。
- 出品文テンプレの「｜天然木」の部分は、AIが素材を判定できた場合はその素材名（オーク無垢／ファブリック等）に自動で差し替わります。判定できなかった場合のみ「天然木」を表示します（誤った素材表記を避けるための実装上の調整です）。
- 着払い判定・出品文生成・査定サマリー・利益シミュレーションはテンプレ／表示ロジックのみで完結しており、外部APIを追加で呼んでいません（追加コストなし）。
- Gemini・Cloud Visionのコードは `pipeline.py` に残していますが、現在アプリからは呼び出していません（Claude固定）。将来チャネル/エンジンを見直す際の材料です。
